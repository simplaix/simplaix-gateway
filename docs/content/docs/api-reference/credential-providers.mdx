---
title: Credential Providers
description: Manage credential providers that define how each credential type works.
---

Credential providers define the authentication type and configuration for each service. All endpoints require JWT authentication.

## Create Credential Provider

```bash
POST /api/v1/credential-providers
Authorization: Bearer <jwt>
Content-Type: application/json

{
  "name": "Gateway API",
  "serviceType": "gateway_api",
  "authType": "jwt",
  "config": {
    "connectUrl": "/auth/connect?service=gateway_api",
    "jwt": { "headerName": "Authorization", "prefix": "Bearer " }
  }
}
```

### Supported Auth Types

| Auth Type | Description | Config Fields |
|-----------|-------------|---------------|
| `oauth2` | OAuth 2.0 flow | `connectUrl`, `oauth2` (client settings) |
| `api_key` | Static API key | `connectUrl`, `apiKey` (header config) |
| `jwt` | JSON Web Token | `connectUrl`, `jwt` (header name, prefix) |
| `basic` | Basic auth | `connectUrl`, `basic` (header config) |

## List Credential Providers

```bash
GET /api/v1/credential-providers
Authorization: Bearer <jwt>
```

## Get Credential Provider

```bash
GET /api/v1/credential-providers/:id
Authorization: Bearer <jwt>
```

## Update Credential Provider

```bash
PUT /api/v1/credential-providers/:id
Authorization: Bearer <jwt>
Content-Type: application/json

{
  "name": "Updated Gateway API",
  "config": { ... }
}
```

## Delete Credential Provider

```bash
DELETE /api/v1/credential-providers/:id
Authorization: Bearer <jwt>
```
